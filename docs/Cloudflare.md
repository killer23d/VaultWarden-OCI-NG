# Cloudflare Integration Guide

This guide provides comprehensive instructions for integrating Cloudflare with your VaultWarden-OCI-NG deployment to enhance security, performance, and reliability.

## Why Use Cloudflare?

Integrating Cloudflare offers several significant advantages:
- **DDoS Protection**: Shields your server from Distributed Denial of Service attacks.
- **Web Application Firewall (WAF)**: Filters malicious traffic and common web vulnerabilities.
- **Hides Your Server's IP Address**: Prevents attackers from directly targeting your server.
- **Content Delivery Network (CDN)**: Caches static assets to speed up load times for users.
- **API-Level Intrusion Prevention**: Allows Fail2ban to block attackers at the network edge, before they even reach your server.

## Prerequisites

- A registered domain name.
- A free or paid Cloudflare account.
- Your domain's nameservers pointing to Cloudflare.

## Step-by-Step Integration

### Step 1: DNS Configuration

In your Cloudflare dashboard, configure the DNS record for your VaultWarden domain.

1.  Navigate to your domain's **DNS** settings.
2.  Add an **A record**:
    *   **Type**: `A`
    *   **Name**: Your chosen subdomain (e.g., `vault` for `vault.yourdomain.com`).
    *   **IPv4 address**: Your server's public IP address.
    *   **Proxy status**: Ensure it is **Proxied** (orange cloud). This is crucial for hiding your IP and enabling Cloudflare's security features.

  *Image is for illustrative purposes.*

### Step 2: SSL/TLS Configuration

Proper SSL/TLS configuration is essential for end-to-end encryption.

1.  Navigate to **SSL/TLS** > **Overview**.
2.  Set your encryption mode to **Full (Strict)**.

    *   **Why is this critical?**
        *   **Off**: No encryption.
        *   **Flexible**: Encrypts traffic from the user to Cloudflare, but **not** from Cloudflare to your server. This is insecure.
        *   **Full**: Encrypts the entire channel but doesn't verify the certificate on your server.
        *   **Full (Strict)**: Encrypts the entire channel and verifies the Let's Encrypt certificate automatically generated by Caddy on your server. This is the most secure option.

3.  Navigate to **SSL/TLS** > **Edge Certificates**.
4.  Enable **Always Use HTTPS** to automatically redirect all `http` requests to `https`
5.  Enable **Automatic HTTPS Rewrites** to help fix mixed content issues.

### Step 3: Fail2ban Integration

This is one of the most powerful features of the VaultWarden-OCI-NG stack. By providing a Cloudflare API token, you allow Fail2ban to block malicious IPs at Cloudflare's edge.

1.  **Generate a Cloudflare API Token**:
    *   In the Cloudflare dashboard, go to **My Profile** > **API Tokens**.
    *   Click **Create Token**.
    *   Use the **Edit zone firewall** template.
    *   Configure the permissions:
        *   **Permissions**:
            *   `Zone` -> `Zone` -> `Read`
            *   `Zone` -> `Firewall Services` -> `Edit`
        *   **Zone Resources**:
            *   `Include` -> `Specific zone` -> Your domain name.
    *   Create the token and copy the generated value. **You will only see this value once.**

2.  **Add the API Token to Your Secrets**:
    Use the `edit-secrets.sh` script to securely add your new token.

    ```
    sudo ./tools/edit-secrets.sh
    ```

    In the editor, add or update the `cloudflare_api_token` value:

    ```
    # secrets/secrets.yaml
    admin_token: "your-admin-token"
    smtp_password: "your-smtp-password"
    backup_passphrase: "your-backup-passphrase"
    cloudflare_api_token: "your-paste-cloudflare-api-token-here"
    ```
    Save and close the file. The secrets will be automatically re-encrypted.

3.  **Restart the Stack to Apply Changes**:
    The new secret will be loaded on the next startup.

    ```
    ./startup.sh
    ```

    Fail2ban will now automatically use this token to manage firewall rules in your Cloudflare account.

### Step 4: Real IP Restoration

Because all traffic is proxied through Cloudflare, your server's logs will show Cloudflare's IP addresses instead of the actual visitor's IP. The VaultWarden-OCI-NG stack handles this automatically.

*   The `startup.sh` script runs `tools/update-cloudflare-ips.sh` on every start.
*   This script fetches Cloudflare's current IP ranges and configures Caddy to trust the `X-Forwarded-For` header from these IPs.
*   This ensures that both Caddy and VaultWarden logs show the correct client IP address, which is essential for effective Fail2ban monitoring.

No manual action is required for this step.

## Troubleshooting Common Issues

### Infinite Redirect Loops
*   **Symptom**: The browser reports `ERR_TOO_MANY_REDIRECTS`.
*   **Cause**: This almost always happens if your Cloudflare SSL/TLS mode is set to **Flexible**. Caddy tries to redirect to `https`, and Cloudflare tries to redirect to `http`, creating a loop.
*   **Solution**: Change the SSL/TLS encryption mode in Cloudflare to **Full (Strict)**.

### Fail2ban Not Blocking IPs in Cloudflare
*   **Symptom**: You see ban actions in Fail2ban logs, but no IPs are added to the Cloudflare firewall.
*   **Causes and Solutions**:
    1.  **Incorrect API Token**: Regenerate the token and ensure it has the correct permissions (`Zone:Read` and `Firewall Services:Edit`).
    2.  **Incorrect Zone ID**: The scripts automatically detect the Zone ID, but if this fails, you may need to specify it. Check the Fail2ban logs for errors.
    3.  **Token Not Loaded**: Ensure you have restarted the stack (`./startup.sh`) after adding the `cloudflare_api_token` to your secrets.

### SSL/TLS Handshake Errors (Error 525)
*   **Symptom**: Browser shows a Cloudflare "Error 525: SSL handshake failed" page.
*   **Cause**: Cloudflare cannot establish a secure connection to your server. This usually means the Caddy service is down or its SSL certificate is invalid.
*   **Solution**:
    1.  Check the status of the Caddy container: `docker compose ps`
    2.  Review Caddy logs for certificate errors: `docker compose logs caddy`
    3.  Ensure your server's firewall is not blocking traffic on ports 80 and 443 from Cloudflare's IP ranges.

By following this guide, you can leverage Cloudflare to significantly improve the security and performance of your self-hosted VaultWarden instance.
